# CFRC Pipeline 前处理工具

把任意几何（STL 或 SIMP 拓扑优化结果）转成 Abaqus 可直接计算的等大六面体网格，跑完应力分析后回收为 `topo_stress_result.mat` 喂给现有切片/路径规划管线。

---

## 文件清单

| 文件 | 作用 | 运行环境 |
|---|---|---|
| **`voxelize.py`** | 统一前端：STL 或 SIMP mat → `.inp` + `.npz` | Python 3（trimesh + scipy） |
| **`voxelize.m`** | MATLAB 包装器 | MATLAB（背后调 Python 3） |
| **`abaqus_odb_to_mat.py`** | Abaqus 跑完后：ODB → `topo_stress_result.mat` | Abaqus 2021 (Python 2.7) |

---

## 完整工作流

```
                  ┌──────────────────┐                     ┌────────────────────────┐
                  │   STL 几何文件    │                     │  SIMP 拓扑优化 .mat     │
                  │   (实心结构)      │                     │  (含 xPhys 连续密度)    │
                  └────────┬─────────┘                     └────────┬───────────────┘
                           │                                          │
                  voxelize.py from-stl              voxelize.py from-topo
                           │                                          │
                           └──────────────────┬───────────────────────┘
                                              │
                              ┌───────────────▼──────────────────┐
                              │  voxel_grid.inp  +  voxel_grid.npz │
                              │  (Abaqus 可直接 Import)            │
                              └───────────────┬──────────────────┘
                                              │
                              在 Abaqus/CAE 里加 BC + Load + Step + Job
                                              │
                                              ▼
                                        job.odb
                                              │
                              abaqus python abaqus_odb_to_mat.py --odb ... --npz ...
                                              │
                                              ▼
                                topo_stress_result.mat
                              （STL 模式: xPhys=0/1 掩码;
                               Topo 模式: xPhys=连续密度;
                               t_xoy / t_xoz: 主应力方向）
                                              │
                                              ▼
                       voxel_refinement_from_test.m → ... → 路径生成
```

## Step 1：生成 Abaqus 输入文件

### 装依赖

```bash
pip install trimesh numpy scipy manifold3d
# v7.3 mat 文件支持(可选):
pip install h5py
```

### STL 模式

```bash
# 体素边长 1.0 (与 STL 同单位)
python3 voxelize.py from-stl part.stl -s 1.0

# Y-up STL (SolidWorks 默认), 需要旋转
python3 voxelize.py from-stl part.stl -s 1.0 -r y-to-z

# 自定义材料 + 输出
python3 voxelize.py from-stl blade.stl -s 0.5 -E 110000 -v 0.34 \
    -i blade.inp -m blade.npz
```

MATLAB:

```matlab
voxelize('stl', 'part.stl', 'VoxelSize', 1.0);
voxelize('stl', 'part.stl', 'VoxelSize', 1.0, 'RotateAxis', 'y-to-z');
```

### 拓扑优化模式

直接吃 SIMP 优化输出的 mat 文件（必须含 `xPhys` 字段，可选 `nelx/nely/nelz/Iter`）：

```bash
# 默认阈值 0.5
python3 voxelize.py from-topo simp_result.mat

# 自定义阈值 + 体素边长
python3 voxelize.py from-topo simp_result.mat -s 1.0 -t 0.4

# 自定义输出
python3 voxelize.py from-topo simp_result.mat -i topo.inp -m topo.npz
```

MATLAB:

```matlab
voxelize('topo', 'simp_result.mat');
voxelize('topo', 'simp_result.mat', 'DensityThreshold', 0.4);
```

#### Topo 模式的关键行为

- 网格尺寸从 mat 文件的 `xPhys` shape 自动推断（`(nely, nelx, nelz)`）
- 仅 `xPhys > threshold` 的体素写入 .inp（变成 Abaqus 单元）
- **完整连续密度场保留在 npz 里**，后续 `abaqus_odb_to_mat.py` 直接用作 `topo_stress_result.mat` 的 `xPhys`（保留原始 SIMP 优化结果，不被二值化破坏）
- 支持 MATLAB v5、v7 mat 文件。v7.3 (HDF5) 需要装 `h5py`，或在 MATLAB 用 `save(..., '-v7')` 重存

### 生成的 .inp 包含什么

- `*NODE`：每个体素角点，共享面节点合并
- `*ELEMENT, TYPE=C3D8R`：每个有效体素一个单元
- `*NSET`：6 个边界节点集 `N_XMIN`/`N_XMAX`/`N_YMIN`/`N_YMAX`/`N_ZMIN`/`N_ZMAX` 供 BC 拾取
- `*SOLID SECTION` + `*MATERIAL`：线弹性占位

**不包含**：Step, BC, Load, Job —— 你在 CAE 里自己加。

### 坐标轴对齐

切片沿 Z 轴进行（`xPhys` 的 `nelz` 维就是切片维）。STL 必须 **Z-up**。9 种旋转预设：

`y-to-z`, `z-to-y`, `x-to-z`, `z-to-x`, `x-to-y`, `y-to-x`, `flip-x`, `flip-y`, `flip-z`

旋转在体素化前应用，全链条（Abaqus / ODB / mat / 切片）使用统一坐标系。

---

## Step 2：Abaqus 加 BC / Load / Job

1. 启动 Abaqus/CAE
2. **File → Import → Model → voxel_grid.inp**（自动加载 Part / Material / Section / Assembly）
3. Step 模块：创建 Step (`Static, General`)
4. Load 模块：加 BC 时 Region 选 `Sets`，从下拉里选 `PART-1.N_XMIN` 等
5. 加 Load（集中力 / 分布压力等）
6. Job 模块：创建并 Submit
7. 完成后得到 `job.odb`

---

## Step 3：ODB → topo_stress_result.mat

```bash
abaqus python abaqus_odb_to_mat.py --odb job.odb --npz voxel_grid.npz --output topo_stress_result.mat
```

> Windows cmd 用户：不要用 `\` 续行，那是 bash 语法。要么写一行，要么 cmd 用 `^` / PowerShell 用反引号 `` ` ``。

### 行为根据 npz 自动切换

| npz 字段 `source_mode` | xPhys 来源 | 用途 |
|---|---|---|
| `'stl'` | 占用矩阵，0 或 1 | 实心结构应力分析 |
| `'topo'` | npz 里保存的连续密度场 | 保留 SIMP 优化结果 |

`t_xoy` 和 `t_xoz` 来自每个单元中心的应力张量（第一主应力方向），公式与 `voxel_refinement_from_test.m` 一致：

```
u = cos(t_xoz)·cos(t_xoy)
v = cos(t_xoz)·sin(t_xoy)
w = sin(t_xoz)
=> t_xoy = atan2(vy, vx) [deg],  t_xoz = atan2(vz, sqrt(vx²+vy²)) [deg]
```

### 可选参数

```
--frame INT          帧索引 (默认 -1, 最后一帧)
--principal MODE     主应力选择: max (默认), tension, compression
--instance NAME      指定实例 (默认自动)
--step NAME          指定 Step (默认最后一个)
```

### 输出 .mat 字段（与 voxel_refinement_from_test.m 完全对齐）

| 字段 | 形状 | 含义 |
|---|---|---|
| `nelx`, `nely`, `nelz`, `nele` | 标量 | 网格尺寸 |
| `xPhys` | `[nely, nelx, nelz]` | 密度场（按模式决定语义） |
| **`mask`** | **`[nely, nelx, nelz]`** | **二值有效单元掩码（1 = 进入 Abaqus 计算）** |
| `t_xoy` | `[nely, nelx, nelz]` | 方位角，度；**掩膜外位置恒为 0** |
| `t_xoz` | `[nely, nelx, nelz]` | 仰角，度；**掩膜外位置恒为 0** |
| `Iter` | 标量 | 帧编号 |

调试元字段：`voxel_size`、`origin_xyz`、`source_mode`、`xPhys_semantics`、`source_odb`、`source_npz`、`principal_mode`、`density_threshold`（topo 模式）、`Iter_history`（topo 模式有迭代历史时）。

## 材料掩膜语义（STL vs Topo 的核心区别）

这是两种输入模式最本质的差异，理解它能避免大量混淆：

|  | STL 模式 | Topo 模式 |
|---|---|---|
| 输入 | 几何形状 | 带网格的连续密度场 |
| 网格来源 | **体素化**生成 | 优化器已经给定 |
| 有效单元判据 | "几何体内部" | **`xPhys > threshold`** |
| 写入 .inp 的单元 | 所有 occupied 体素 | 仅 `xPhys > threshold` 的单元 |
| 输出 mat 中的 `xPhys` | 0/1 掩码（=mask） | **连续密度场（保留 SIMP 原值）** |
| 输出 mat 中的 `mask` | 同 xPhys，纯冗余 | **明确的二值掩码** |
| 输出 mat 中的 `t_xoy/t_xoz` | 所有体素都有应力数据 | **掩膜外位置 = 0**（无应力数据） |

**关键不变量**：在 Topo 模式下 `mask == (xPhys > threshold)`，所以下游脚本可以二选一：
- 沿用旧约定：`xPhys > DENSITY_THRESHOLD`
- 直接用新字段：`mask > 0.5`

两种写法语义等价。`mask` 字段的好处是消除"threshold 取多少"的不确定性——voxelize 时用的 threshold 已经"烧"进 mask 里了。

**为什么要保留连续密度场而不是直接二值化？**  
下游切片脚本（`generate_reference_surface.m` 等）可能用密度梯度做插值、加权或其他操作。直接二值化会丢失这些信息。我们做的是：用掩膜决定**哪些单元参与 Abaqus 应力计算**，但**密度数值原样保留**。

## 常见坑

**Q1：`abaqus_odb_to_mat.py` 报 unrecognized arguments `\`**  
A：Windows cmd 不认 bash 续行符。写一行命令，或用 `^` (cmd) / `` ` `` (PowerShell)。

**Q2：拓扑优化 mat 是 v7.3 格式，scipy.io 读不了**  
A：装 `pip install h5py`，或在 MATLAB 用 `save('result.mat', 'xPhys', 'nelx', 'nely', 'nelz', '-v7')` 重存。

**Q3：from-topo 报 "No density field found"**  
A：脚本依次找 `xPhys` / `x` / `rho` / `density`。如果你的字段名不在这里，先在 MATLAB 里改名重存。

**Q4：1D 密度向量怎么处理**  
A：脚本会自动 reshape 到 `(nely, nelx, nelz)`，但**必须**在 mat 里同时有 `nelx`/`nely`/`nelz`（标量字段），否则报错。

**Q5：单元数太多，Abaqus 求解慢**  
A：从源头减少。STL 模式：增大 `--voxel-size`。Topo 模式：提高 `--density-threshold`，让更多体素被排除。

**Q6：N_XMIN 等节点集太大，CAE 卡**  
A：上千节点的 NSET 加 BC 完全没问题。如果你想要更小的 BC 集（比如只固定一个角），可在 CAE 里手动 Create Set，或在 .inp 里手动加 `*NSET, NSET=MY_CORNER`。

**Q7：模型有自相交，体素化失败**  
A：trimesh 的 `process(validate=True)` 会尝试修复，严重自交需要 MeshLab/Meshmixer 预处理。或 `--no-fix` 看原始失败原因。

**Q8：没装 Abaqus 也能跑应力吗**  
A：可以用 [CalculiX](http://www.calculix.de/) `ccx`，吃同一套 .inp。Step/BC/Load 写法略不同，自己编辑 .inp。`ccx` 输出 .frd 需自行解析（不在本工具集里）。

## 关键设计决策

**为什么 .inp 不带 Step/BC/Load？**  
每个工程问题的 BC 和 Load 都不一样。在 CAE 里拖拽设置比让脚本猜工况靠谱。预置的 6 个 NSET 已让设置非常快。

**为什么用 C3D8R 不用 C3D8？**  
C3D8 全积分有 volumetric locking 风险（尤其 ν → 0.5）。C3D8R 缩减积分稳定且快。CLI 三种都支持。

**为什么 Topo 模式把连续密度场塞 npz 里？**  
.inp 文件只能描述网格几何（"哪些体素存在"），但拓扑优化的连续密度信息是后续切片需要的物理量。把它存在 npz 里跟应力一起带到最后的 mat，避免信息丢失，也避免 `abaqus_odb_to_mat.py` 还得回头读 SIMP 的 mat。

**为什么 t_xoz 用 +asin(dz) 而不是 -asin(dz)？**  
匹配你 `step3_extract_stress.py` 一直在用的约定。之前我误把 `voxel_refinement_from_test.m` 的方向公式当作 `w = -sin(t_xoz)`，实际应为 `w = +sin(t_xoz)`。已修正。
